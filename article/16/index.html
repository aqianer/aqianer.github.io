Spring Boot 应用 CICD 集成 Jenkins 复盘文档

一、目标概述

将简单的 Spring Boot 应用通过 Jenkins 实现 持续集成与持续部署（CICD），实现代码提交后自动编译、测试、打包并部署到华为云服务器。

2. Java 版本不一致与优先级冲突

  • 原因：Linux 中 环境变量硬编码（JAVA_HOME）优先级高于 update-alternatives 动态配置。

（二）Day 2：Jenkins 插件与 CICD 配置

1. Jenkins 插件安装失败（镜像源与网络配置）

  • 问题现象：首次安装推荐插件时大量失败，提示“无法连接镜像源”。
  • 原因分析：
    • 清华镜像源 https://mirrors.tuna.tsinghua.edu.cn/jenkins/updates/update-center.json 资源 404（镜像站路径变更）。
    • 云服务器安全组/防火墙限制出站请求。
  • 解决方案：
    1. 更换镜像源：将 Jenkins 升级站点 URL 替换为华为云镜像源 https://mirrors.huaweicloud.com/jenkins/updates/update-center.json。
    2. 检查网络配置：
      • 本地防火墙：执行 sudo ufw status，若为 inactive 则默认放行所有出站请求；若为 active，需添加规则 sudo ufw allow out to mirrors.huaweicloud.com。
      • 云服务器安全组：在华为云控制台“安全组规则”中，确保出方向规则允许 80/443 端口访问公网。

2. Git 与 Maven 配置

  • Git 配置：
    • 需在 Jenkins 中添加 GitHub 凭证（Personal Access Token，PAT）：进入“凭证管理”→“添加凭证”，选择“Username with password”，输入 GitHub 用户名和 PAT。
  • Maven 配置：
    • 在 Jenkins“全局工具配置”中，选择“自动安装”Maven，版本与本地开发环境（如 3.5.3）保持一致，确保编译环境统一。

3. SSH 远程部署配置

  • 问题现象：通过 SSH 部署到目标服务器时权限不足或连接失败。
  • 解决方案：
    1. 生成 SSH 密钥对：在 Jenkins 服务器执行 ssh-keygen -t rsa，生成公钥 id_rsa.pub 和私钥 id_rsa。
    2. 配置目标服务器信任：将公钥 id_rsa.pub 追加到目标服务器的 ~/.ssh/authorized_keys 文件中。
    3. 在 Jenkins 中配置 SSH 凭证：
      • 进入“凭证管理”→“添加凭证”，选择“SSH Username with private key”，输入目标服务器用户名，粘贴私钥内容。
    4. 测试 SSH 连通性：在 Jenkins 任务中配置“Publish Over SSH”，选择上述凭证，填写目标服务器 IP、端口（默认 22），点击“Test Configuration”验证连通性。
    5. 权限调整：确保目标服务器上 Jenkins 工作目录的属主为 jenkins 用户（而非 root），执行 sudo chown -R jenkins:jenkins /path/to/workdir。

三、总结与经验沉淀

  1. 技术细节把控：

    • Spring Boot 打包需显式配置 spring-boot-maven-plugin，否则生成的 Jar 包不可执行。
    • Java 版本管理需注意环境变量优先级高于 update-alternatives 动态配置，需优先清理硬编码的 JAVA_HOME 和 PATH。

  2. Jenkins 运维要点：

    • 密码丢失时，重装是最可靠的兜底方案，重装后需重点关注插件镜像源和网络配置。
    • 插件安装失败时，优先排查镜像源有效性和云服务器安全组/防火墙规则，必要时更换国内镜像源（如华为云、Jenkins 中文社区）。

  3. CICD 流程落地：

    • Git 凭证需使用 GitHub PAT 确保权限安全，Maven 版本需与本地开发环境对齐。
    • SSH 部署需通过密钥对认证保障安全，同时注意目标服务器的文件权限归属。

通过本次复盘，明确了从 Spring Boot 应用开发到 Jenkins CICD 落地的全流程痛点与解决方案，后续可将这些经验沉淀为团队 CICD 规范，提升同类项目的交付效率。
